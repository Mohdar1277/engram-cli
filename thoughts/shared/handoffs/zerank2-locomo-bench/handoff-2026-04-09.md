### Lifecycle retry hardening (commits `3636a0c`, `cf64c28`, gemini retry, `9b08bb0`, `4eccb0d`)
EVERY API client engram talks to now has identical retry coverage. This was the root cause of multiple bench failures during the session — at 1500+ sequential requests over 40-60 minutes, every upstream provider hits at least one transient. The pattern:
- Retries: send / 429 / 5xx / body read / JSON parse → up to 6 attempts with exponential backoff `[2,4,8,16,32,64]s` (or `[1,2,4,8,16,32]s` for localhost zerank_local where upstream recovers faster)
- 4xx (non-429) stays permanent — never retry real client errors
- Bench output is transactional: `run_*_qa` returns `Err` only after all retries exhaust, and the report file is never written until the report is fully built

Files with retry now:
- `crates/engram-llm/src/openrouter.rs` (commits 3636a0c + cf64c28)
- `crates/engram-embed/src/gemini.rs` (`embed_one()` had ZERO retry; `embed_batch()` had partial)

## Gotchas & Warnings

### zerank-2 specific
- **Pin `transformers<5.0`** when launching the sidecar. transformers 5.x broke `_tied_weights_keys` when it's a list ([huggingface/transformers#44861](https://github.com/huggingface/transformers/issues/44861)) and zerank-2's modeling code uses the older API. Symptom: `AttributeError: 'list' object has no attribute 'keys'` at model load.
- **Don't use `sentence_transformers.CrossEncoder.predict()`** with zerank-2. It calls the tokenizer with single texts, but `ZeroEntropyTokenizer.__call__` expects a list of (query, doc) pairs. Symptom: `too many values to unpack (expected 2)`. Use the direct `AutoTokenizer + AutoModelForSequenceClassification` path instead.
- **Use `AutoModelForSequenceClassification`, NOT `AutoModelForCausalLM`.** The custom modeling code registers the class as `ZeroEntropyForSequenceClassification`. Symptom: `Unrecognized configuration class ... for AutoModelForCausalLM`.
- **The model returns `[batch, 1]`, not `[batch, seq_len, vocab]`.** zerank-2's forward() already extracts the yes-token logit at the last position internally and divides by 5 for calibration. Just `.squeeze(-1)` the output. Don't try to do manual yes-token extraction. Symptom: `too many indices for tensor of dimension 2`.
- **`torch_dtype` is deprecated, use `dtype=`** (transformers 4.46+).
- **The sidecar uses port 8765 hard-coded.** Override with `ENGRAM_ZERANK_URL=http://...` env var if needed.
- **The sidecar uses `mps` (Metal) automatically on Apple Silicon.** Falls back to cuda or cpu otherwise.
- **Threading server can drop connections under contention.** When running parallel benches, the localhost listen backlog can fill briefly. The Rust client retries automatically (commit `9b08bb0`).

### LoCoMo bench specific
- **Categories are 1, 2, 3, 4, 5 in the data — but our 50-question sample only had 1, 2, 3.** The 50 was the first 50 QAs of sample 0, and didn't include category 4 (open-domain knowledge, 841 questions out of 1542 with gold) or category 5 (adversarial, 446). The full 1542-question run will surface category 4 for the first time, and the score may drop substantially because category 4 is the largest single bucket and is harder.
- **The user-visible 82% number is from the 50-Q biased sample.** Don't claim engram is at 82% on full LoCoMo until the full bench finishes.
- **Mem0 SOTA is 66.88% on full LoCoMo with GPT-4o-mini answerer.** Our full bench uses gpt-5.4-mini (newer/better), so even matching Mem0 on architecture would beat them on the answerer axis. Apples-to-oranges.
- **The bench writer is transactional**: it ONLY writes the report file on full success. Failed runs leave NO partial state. This is intentional per the user's instruction "make sure that our progress is not saved if there's an error".

### Retry behavior
- **Every API client now has lifecycle retry.** This includes Gemini embed (single + batch), OpenRouter chat, Cohere rerank, zerank_local rerank. If a NEW API client is added in the future, port the same retry pattern from `engram-rerank/src/cohere.rs` or `engram-rerank/src/zerank_local.rs`.
- **The retry covers send / 429 / 5xx / body read / JSON parse.** 4xx (non-429) is permanent — real client errors should not be retried.
- **Localhost retries use `[1,2,4,8,16,32]s` backoff, remote APIs use `[2,4,8,16,32,64]s`.** Both 6 attempts max.
